Help

Here are the configuration parameters for all available elements.

Every element has to have a name (can be blank but what is the point then) which will represent the key value in json.

JSON_BOOL

Adds option to generate boolean values.

only TRUE
mark that if you want to only generate true values
only FALSE
mark that if you want to only generate false values

WARNING: marking both only TRUE and only FALSE is an error.

JSON_DATE

Adds option to generate date values.

lower bound
all the generated dates will not be before the given value
upper bound
all the generated dates will not be after the given value
output pattern
specifies the format in which the date will be written. Has to follow rules of java.time.format.DateTimeFormatter

If you want to generate dates on a specified day set lower bound equal to upper bound.

WARNING: lower bound must not be after upper bound.

JSON_DOUBLE

Adds option to generate numerical values with decimal part.

lower bound
all the generated numbers will not be smaller than the given value
upper bound
all the generated dates will not be greater than the given value
precision
specifies the count of numbers after the decimal point

WARNING: lower bound must be lower than upper bound and precision has to be at least 1.

JSON_NUMBER

Adds option to generate numerical values with 0 places after decimal point.

lower bound
all the generated numbers will not be smaller than the given value
upper bound
all the generated dates will not be greater than the given value

WARNING: lower bound must not be greater than upper bound.

JSON_OBJECT

A container that allows to create a nested json that consists of all available values.

count
specifies format of the nested json. For 1 it will be a single object, for more than 1 it will be an array of objects.

You can nest another json in already nested json. Use the JSON_OBJECT when adding new elements.

JSON_PATTERN

Adds option to generate string values that are formatted in given pattern.

pattern
specifies the pattern of the string; the pattern is of given format: number-number-number, i.e. 3-4-12
connector
specifies the element that will connect the given parts; can be one symbol or many, or even empty
all capital
marking that option will capitalize the given output

WARNING: pattern must start with a number but can be as short as one number.

JSON_STRING

Adds option to generate string values.

min length
specifies the minimum length of generated string
max length
specifies the maximum length of generated string
first capital
margin that option will capitalize only the first letter
all capital
marking that option will capitalize the given output

WARNING: min length cannot be greater than max length.

WARNING: marking both first capital and all capital is an error.
